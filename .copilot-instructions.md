# API Data Fetching & React Component Integration

## Project Architecture

This is a **TypeScript React (Vite) + Axios + Spring Boot Backend** project.

**Data Flow:**
```
React Component → useApi Hook → API Service → Axios Client → Backend API
```

## Key Files & Their Purposes

| File | Purpose | What to Do |
|------|---------|-----------|
| `src/apiClient.ts` | Axios instance with auth interceptors | Don't modify - already configured |
| `src/apiService.ts` | All API endpoint methods | Call methods from here (e.g., ProjectService.getAllProjects) |
| `src/hooks/useApi.ts` | Custom hook for async state management | Use this hook in all data-fetching components |
| `src/types.ts` | TypeScript interfaces for all DTOs | Import types for type safety |

## Basic Pattern (5 Steps)

### Step 1: Import
```typescript
import useApi from '../hooks/useApi';
import { ProjectService } from '../apiService';
import { ProjectDto } from '../types';
```

### Step 2: Use Hook
```typescript
const { data, loading, error, execute } = useApi(ProjectService.getAllProjects);
```

### Step 3: Fetch on Mount
```typescript
useEffect(() => {
  execute();
}, [execute]);
```

### Step 4: Handle States
```typescript
{loading && <div>Loading...</div>}
{error && <div>Error: {error.message}</div>}
{data && <div>Display data here</div>}
```

### Step 5: Render
```typescript
{data?.map(item => <li key={item.id}>{item.name}</li>)}
```

## Available API Services

### ProjectService
- `getAllProjects()` → `ProjectDto[]`
- `getProjectById(id)` → `ProjectDto`
- `createProject(data)` → `ProjectDto`
- `updateProject(id, data)` → `ProjectDto`
- `deleteProject(id)` → `void`

### SprintService
- `getSprintsByProject(projectId)` → `SprintDto[]`
- `createSprint(projectId, data)` → `SprintDto`
- `updateSprint(projectId, sprintId, data)` → `SprintDto`
- `deleteSprint(projectId, sprintId)` → `void`

### TaskService
- `getTasksByProject(projectId)` → `TaskDto[]`
- `createTask(data)` → `TaskDto`
- `updateTask(taskId, data)` → `TaskDto`
- `deleteTask(taskId)` → `void`

### AuthService
- `register(data)` → `User`
- `authenticate(credentials)` → `AuthenticationResponse`

## Common Use Cases

### Display a List
```typescript
function MyList() {
  const { data, loading, error, execute } = useApi(ProjectService.getAllProjects);
  
  useEffect(() => { execute(); }, [execute]);
  
  return (
    <div>
      {loading && <p>Loading...</p>}
      {error && <p>Error: {error.message}</p>}
      {data?.map(item => <div key={item.id}>{item.name}</div>)}
    </div>
  );
}
```

### Fetch by URL Parameter
```typescript
function Detail() {
  const { id } = useParams();
  const { data, loading, error, execute } = useApi(ProjectService.getProjectById);
  
  useEffect(() => {
    if (id) execute(Number(id));
  }, [id, execute]);
  
  return <div>{data?.projectName}</div>;
}
```

### Form Submission
```typescript
function CreateForm() {
  const [form, setForm] = useState({});
  const { loading, error, execute: create } = useApi(ProjectService.createProject);
  
  const handleSubmit = async (e) => {
    e.preventDefault();
    await create(form);
    // Reset form or navigate
  };
  
  return <form onSubmit={handleSubmit}>...</form>;
}
```

### Delete with Confirmation
```typescript
async function handleDelete(id) {
  if (confirm('Are you sure?')) {
    await ProjectService.deleteProject(id);
    // Refresh list or navigate
  }
}
```

### Cascading Fetches
```typescript
function Combined() {
  const { data: project, execute: getProject } = useApi(ProjectService.getProjectById);
  const { data: sprints, execute: getSprints } = useApi(SprintService.getSprintsByProject);
  
  // Fetch project first
  useEffect(() => {
    getProject(id);
  }, [id, getProject]);
  
  // Then fetch sprints
  useEffect(() => {
    if (project?.projectID) {
      getSprints(project.projectID);
    }
  }, [project?.projectID, getSprints]);
  
  return <div>{sprints?.map(s => <div key={s.id}>{s.name}</div>)}</div>;
}
```

## Error Handling

Always show user-friendly error messages:
```typescript
{error && (
  <div className="bg-red-50 border border-red-200 p-4 rounded">
    <p className="text-red-800">{error.message}</p>
  </div>
)}
```

## Performance Tips

1. **Debounce search:** Wait 300-500ms after user stops typing before API call
2. **Avoid re-fetches:** Check if data is already loaded before calling execute
3. **Lazy load:** Only fetch when user navigates to section
4. **Pagination:** Load data in chunks instead of all at once

## Dos & Don'ts

✅ **DO:**
- Always handle loading, error, empty states
- Use useEffect to trigger API calls
- Import types for type safety
- Show user-friendly error messages
- Debounce search/filter calls

❌ **DON'T:**
- Don't call execute() in render
- Don't skip execute from useEffect dependencies
- Don't make API calls in loops
- Don't expose raw errors to users
- Don't forget to handle null/undefined data
